SOURCE CODE MODULES
================================================================================

src/metrics.py
   Clustering evaluation metrics used throughout the project:
   
   Label-aware metrics:
   - ACC (Accuracy)              : Hungarian matching for label assignment
   - NMI (Normalized Mutual Info): Mutual information normalization
   - ARI (Adjusted Rand Index)   : Corrected Rand index for randomness
   
   Label-free metrics:
   - Silhouette Score     : Cohesion vs separation (range: -1 to 1)
   - Davies-Bouldin Index : Cluster separation quality (lower is better)
   - Calinski-Harabasz    : Ratio of between-cluster to within-cluster variance

src/model.py
   Core GEDI implementation and clustering utilities:
   
   Key classes/functions:
   - GEDIConfig           : Hyperparameter dataclass
   - GEDIModel            : nn.Module with encoder → projector → cluster centers
   - train_gedi()         : Full training loop with energy-based learning
   - gedi_predict()       : Hard cluster assignment from trained model
   - run_clustering_suite(): Compare GEDI with sklearn baselines
   - run_ablation_study()  : Analyze each loss component contribution

   Baseline algorithms available:
   - KMeans
   - Spectral Clustering
   - Agglomerative Hierarchical
   - Gaussian Mixture Model

src/utils.py
   Utility functions for data and experiment management:
   
   - set_random_seed()              : Fix randomness (seed=42)
   - generate_synthetic_data()      : Create moons/circles datasets
   - load_svhn_dataset()            : Load SVHN image data
   - load_cifar10_dataset()         : Load CIFAR-10
   - load_cifar100_dataset()        : Load CIFAR-100
   - load_fashionmnist_dataset()    : Load Fashion-MNIST
   - get_paper_reference_scores()   : Retrieve Table 3 values from paper
   - preprocess_images()            : Normalize and resize images

✓ ALGORITHM OVERVIEW
   
   GEDI combines three learning objectives:
   
   1. L_INV (Invariance Loss)
      - Ensures predictions robust to data augmentations
      - Small augmentation → similar cluster assignments
      - Loss: Contrastive between augmented pair representations
   
   2. L_PRIOR (Prior Loss)
      - Encourages balanced cluster sizes
      - Prevents collapse to single cluster
      - Loss: KL divergence from uniform cluster distribution
   
   3. L_GEN (Generative/Energy-Based Loss)
      - Core energy-based learning via Stochastic Gradient Langevin Dynamics (SGLD)
      - Models energy function E(x,k) = -log p(k|x)
      - Samples from buffer and matches model distributions
      - Loss: Contrastive divergence between data and model samples
   
   Final loss: L_total = w_inv * L_INV + w_prior * L_PRIOR + w_gen * L_GEN
               (weights typically: 1.0, 0.1, 1.0)

✓ DATA AUGMENTATION STRATEGY
   
   For image clustering:
   - Random rotation (±15°)
   - Random crop (90% zoom)
   - Color jittering (brightness, contrast, saturation)
   - Optional: Gaussian blur, random grayscale
   
   Applied during training to compute L_INV

✓ MODEL ARCHITECTURE
   
   GEDIModel structure:
   
   Input (image)
      ↓
   Encoder (CNN or ResNet-8)
      ↓
   Feature vector (d-dim)
      ↓
   Projector head (2-3 linear layers)
      ↓
   Projected embedding (p-dim, typically < d)
      ↓
   ├─→ Cluster heads → Soft cluster assignments (k classes)
      ├─→ Energy head  → Scalar energy E(x,k)
      └─→ Logits head  → Unnormalized class probabilities
   
   Total parameters: typically 1-5 million (depends on encoder choice)
